    Yet the example given shows an api-paste.ini without any of these settings, and later, shows these settings made in the nova.conf file. This is, so far, the most confusing page I've found in the documentation.

    That line should be removed from the documentation, please disregard it. That content now goes in nova.conf instead of api-paste.ini. Proposed doc fix is here: https://review.openstack.org/1...

  • Bill Borsari

    The example nova.conf here uses the rabbit_host but the setup directions have the user install qpid. The correct statement would be to remove the rabbit_host and replace it with the qpid tag: "qpid_hostname=hostname.example.com"

    to nova.conf.
