    What's purpose to give a region parameter? It seems only to be used when create endpoint.

  • Lorin Hochstein

    The region parameter is useful if you have multiple OpenStack deployments: you can specify the endpoints of each of them in Keystone, using a different region parameter for each one. If you only have a single OpenStack deployment, then the region parameter isn't meaningful.
